# APS Viewer デバッグセッションログ（詳細版）
## 日時: 2026-03-07

### ✅ 開発の軌跡
本プロジェクトの短期間での実装にあたり、Claude Codeをメインエンジニアとして、Geminiをリスクマネージャーとして活用。
以下の技術的課題をAIとの対話を通じて解決しました。

#### 技術的ブレイクスルーのハイライト
* **APS Viewerの描画方式変更**: 
  DataVisualization extensionの不安定さを考慮し、カスタムシェーダーおよびTHREE.jsのOverlay機能を活用した独自ピン描画ロジックへリファクタリング。
* **環境依存エラーの克服**: 
  zsh/TS環境における特殊文字パースエラーを、Pythonスクリプトによるファイル注入方式に切り替えることで解決。

### 📊 技術的注意事項・ナレッジ
| 注意事項 | 内容 |
|---------|------|
| 描画スタック | DataVisualizationを排除しTHREE.js直描画へ。パフォーマンスを優先 |
| 通信制御 | 5秒間隔のポーリング実装による状態の自動同期 |
| 依存性注入 | Application層にPortを定義し、Next.js API Routesから呼び出すDDD構成 |
